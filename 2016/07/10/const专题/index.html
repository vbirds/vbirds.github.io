const专题

const定义

const 是 c/c++中常用的类型修饰符，const修饰的数据类型是指常类型，常类型的变量或对象的值是不能被更新的。

const的作用

const的作用可以分为五种：

  • 定义一个常量
  • const意味着只读
  • 指向常类型数据的指针（所指向的内存数据不能被修改，但本身可以修改）
  • 指向一个指针（指针变量不能被修改，但是它所指向的内存空间可以被修改）
  • 指向一个常类型的常指针（指针和它所指向的内存空间，均不能被修改）

const的好处

  1. 利用const修饰指针做函数参数，可以有效的提高代码可读性，减少bug；
  2. 清楚的分清参数的输入和输出特性

c++的const和c中const的区别

虽然const 在c、c++中的功能差不多，但还是有区别的。c++编译器对const常量做一下处理：

  1. 当碰见常量声明时，在符号表中放入常量（即“=”号后面的值）；
  2. 编译过程中发现使用常量则直接以符号表中的值替换；
  3. 编译过程中若发现对const使用了extern或者&操作符，则给对应的常量分配存储空间（兼容c)

但是要注意的一点是：C++编译器虽然可能为const常量分配空间，但不会使用其存储空间中的值。
所以我们可以得到以下结论：

  1. C语言中const变量是只读变量，有自己的存储空间
  2. c++中的const可能分配存储空间,也可能不分配存储空间 ，当const常量为全局，并且需要在其它文件中使用时不分配空间，使用符号表的值，当使用&操作符取const常量的地址时，编译器会给const分配空间

const和#define的区别

  1. 相同点
    C++中的const常量类似于宏定义如 const int c = 5; ≈ #define c 5
  2. 不同点
    const常量是由编译器处理的，提供类型检查和作用域检查，而宏定义由预处理器处理，单纯的文本替换。

所以在c++里尽量使用const来代替宏定义，这样可以减少程序出错的可能性。
